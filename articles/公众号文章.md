凌晨两点，灵感突然来了。一个绝妙的功能设计在脑子里成型、推演、验证——完美。

但你正躺在床上，手机刷得眼皮打架。爬起来开电脑？算了，明天再说。

然后就没有然后了。第二天坐在工位上，那个灵感就像从来没存在过一样。

或者这样：

周末带孩子去公园，坐在长椅上看着他在沙坑里玩。突然想到一个 bug 怎么修了。掏出手机——嗯，能干点啥呢？刷刷短视频吧。

再或者这样：

出差路上、医院排队、等咖啡……每天有多少"碎片时间"就这么流走了？

如果——我是说如果——掏出手机就能接着写代码，会怎样？

## 市面上不是没有方案

有人会说：用 **OpenClaw** 啊。

对，OpenClaw 很酷。但在实际使用中，**一个会话动辄烧掉几万甚至十几万 token**。对于日常开发来说，这就像用鱼子酱填饱肚子——好吃是好吃，但你扛不住。

还有人说：用 **Hermes Agent** 啊。

Hermes 是 Nous Research 开源的通用 AI Agent 框架，很强大——80k+ GitHub Stars、200+ 模型支持、70+ 工具、持久化记忆。它可以在手机上通过 Termux 运行，听起来很美。

但实际用起来呢？

**要在手机上跑 Hermes Agent，你得先折腾这些：**

1. 装 **Termux**（得从 F-Droid 下，Google Play 的不行）
2. `pkg install python git clang make rust` —— 装一堆编译工具
3. 手动安装 **Python 3.11**（Termux 默认是 3.13，版本不对会翻车）
4. 创建虚拟环境，激活它
5. 处理 **jiter 兼容性** —— ARM 设备上 Rust 编译必挂，你得手动建一个 Python 兼容桩
6. 用 `pip install -e '.[termux]'` 装 Hermes，还得配上约束文件
7. `hermes setup` 交互式配置
8. 再配一通 Telegram/微信/钉钉网关才能从聊天软件里跟它对话

这一套下来，顺利的话 30-60 分钟。不顺利的话...GitHub issues 里翻一翻，全是泪。

**折腾完只是开始，还有几个绕不开的限制：**

| 问题 | 说明 |
|---|---|
| **文件系统隔离** | Hermes 跑在 Termux 的沙箱里，它读写的是手机上的文件，**碰不到你工作机上的项目代码** |
| **交互体验割裂** | 通过 Termux 终端启动 → 再配 Telegram/微信 Gateway 才能用手机聊，消息绕一大圈 |
| **Docker / 浏览器自动化不可用** | Android 没有 cgroups 和 GUI 渲染环境，部分工具直接废掉 |
| **内存开销** | 推荐开低内存模式，手机同时跑其他 App 会吃紧 |
| **不是为写代码设计的** | Hermes 是通用 AI Agent，没有针对 Claude Code 的会话管理、目录选择、Markdown 渲染等开发场景优化 |

说白了一个核心问题：**你用手机写代码，真正干活的是你工作机上的项目和环境。Hermes 跑在手机的 Termux 里，跟你工作机的代码库之间隔了一堵墙。**

还有人用 Termux + tmux + SSH。够折腾，但那个小屏幕上的终端操作体验……只能说充满了极客精神，跟"爽"不沾边。

我们要的不是"能跑就行"，而是**随时随地、掏出手机、自然语言对话、躺着写代码**。

## CC Remote：自建、开源、靠谱

这就是我为什么写了 **CC Remote**。

它是一个两件套系统：

```
┌──────────────┐   WebSocket   ┌────────────────┐    PTY    ┌──────────┐
│  Android APP │ ◄───────────► │  Node.js 服务端 │ ◄───────► │ claude   │
│  (你的手机)   │   局域网/VPN   │  (你的工作站)    │          │ CLI     │
└──────────────┘               └────────────────┘          └──────────┘
```

你的工作机上跑一个 Node.js 服务端，它负责启动 Claude Code CLI。你的 Android 手机通过 WebSocket 连上去，就能：

- **浏览工作机上的所有项目目录**
- **创建、管理多个 Claude Code 会话**
- **用聊天的方式跟 Claude 对话，让它写代码**
- **实时看到 Markdown 回复——表格、代码块、链接全都渲染得好好的**

内核用的是 Anthropic 官方的 Claude Code CLI，token 消耗完全由你自己控制。想精打细算就多用 `claude -p` 单次模式，想深入对话就开完整会话。**不存在"平台抽成"，你付给 Anthropic 的每一分钱都用在自己的代码上。**

代码完全开源（MIT 协议），你甚至可以自己改。

### CC Remote 的优势在哪？

一句话：**Hermes 是跑在手机上的通用 AI Agent，CC Remote 是 Claude Code 的远程遥控器。** 两者做的事完全不同。如果你要的是"用手机遥控工作机上的 Claude Code 写项目代码"，CC Remote 有几个明显的优势：

- **代码和开发环境在工作机上** — 你的项目文件、Git 仓库、node_modules、数据库连接全在工作站上。CC Remote 让手机遥控工作机上的 Claude Code，直接操作真实的开发环境，而不是在手机沙箱里重头搭一套。
- **手机零负担** — 所有计算都在工作站上跑，手机只负责收发消息和显示结果。不费电、不发热、不占内存，跟刷微信一样轻量。
- **原生 App 体验** — 会话列表、聊天界面、Markdown 渲染（表格、代码块、链接）、目录浏览、后台通知，都是正经的 Android 原生 UI，不是终端里敲命令。
- **Claude 干完活了，手机弹通知** — 把 App 切到后台去刷朋友圈？Claude 写完了代码、跑完了测试，手机会弹出通知告诉你。点一下通知直接跳回会话，接着 review（或继续摸鱼）。
- **多项目同时开工** — 可以同时开多个 Claude Code 会话，每个在不同项目目录里干不同的活。A 项目在重构 API，B 项目在写测试，C 项目在修 bug——手机上左右横跳，随时切换查看进度。互不干扰，各自独立。
- **两分钟上手** — 服务端 `npm start`，手机装 APK，填上 IP，完事。不需要折腾 Python 版本、编译工具、虚拟机环境。
- **Web 终端也带着** — 手机不在手边时，浏览器打开也能连同一套会话。
- **token 完全自己掌控** — 直接调用 Claude Code CLI，没有额外的 Agent 框架开销。省下的每一分钱都是实实在在的。**不挑模型** — 你工作机上 Claude Code 能用的所有模型（Claude Sonnet、Opus、DeepSeek 等）在手机上都能遥控，不受手机端限制。

## 说几个真实的用法

### 场景一：Couch Coding

下班回家，吃完饭洗完澡，窝进沙发。突然想到项目里有个重构还没做。

掏出手机 → 打开 CC Remote → 选项目目录 → "帮我把用户模块的 API 调用从 fetch 换成 axios，加上错误重试"

Claude 哐哐哐一顿改。你躺在沙发上看着代码一行行滚动。完事儿了，锁屏睡觉。

### 场景二：Bug 急诊

周末正跟朋友吃饭，手机震了——测试环境的服务挂了。日志看不全，又没带电脑。

打开 CC Remote → 连上服务器 → "看看刚才的 error log，帮我分析一下原因"

Claude 读完日志，定位到是 Redis 连接池泄漏，还顺手给了修复方案。你回了一句"apply"，搞定。

继续吃饭。

### 场景三：碎片时间写代码

通勤地铁上 20 分钟。

打开 CC Remote → 选正在做的 feature → "给这个函数补上单元测试，主要测边界条件"

到站的时候，测试写好了。回家 git push，收工。

## 远程怎么连？

服务端默认监听局域网地址。

在家连 Wi-Fi 直接用。

出门在外？装个 **ZeroTier**（免费支持 25 台设备），手机和工作机加入同一个虚拟局域网，就像在同一间屋子里。**不需要公网 IP、不需要端口转发、不需要暴露服务到公网。**

要是 ZeroTier 被防火墙挡住了，还有 **Tailscale** 可以替代——底层 WireGuard，干净利落。

## 不只是 Android 应用

没带手机？带了个 Chromebook？或者用笔记本的时候也想用？

CC Remote 内置了 **xterm.js Web 终端**。浏览器打开 `http://你的服务器IP:11199/terminal` 就能直接连上 Claude Code 会话。

所以你可以：

- **Android 手机** → 随身携带，刷着 vibe
- **笔记本浏览器** → 坐在星巴克，键盘输入更顺手
- **甚至两者同时连同一个会话** → 跟同事结对编程

- ✅ **多项目同时开发** — 同时跑多个 Claude Code 会话，每个在不同项目目录，互不干扰

- ✅ **多端同时观看** — 多个设备同时看一个会话，手机兜里、电脑 Web 接着聊

## 如何下载和安装？

### 手机端 App（Android）

> **⚠️ 重要提醒：** 手机 APP 只是"遥控器"，你的电脑上必须先安装并运行服务端，手机才能连上。请先设置好电脑端，再装手机 App。

**第 1 步：下载 APK 安装包**

打开手机浏览器，访问 Gitee 发布页：
👉 **[https://github.com/LeisureRain/cc-remote/releases](https://github.com/LeisureRain/cc-remote/releases)**

往下翻，找到最新的版本，点击 **cc-remote-v1.x.x.apk** 下载。（如果你用电脑看的这篇文章，用手机扫一扫下面的码就能直接打开）

> 如果下载时提示"不安全"或"可能有害"，不用慌——因为 APK 不是从应用商店下载的，Android 会弹出警告。点击"仍要下载"或"保留"即可。这是你自己的 APP，没问题的。

**第 2 步：安装 APK**

下载完成后点击通知栏里的文件，或者打开文件管理器找到已下载的 APK：

- **小米/红米（HyperOS/MIUI）：** 点开文件 → 提示"禁止安装未知来源应用" → 去"设置 → 应用 → 系统应用设置 → 安装未知应用"→ 给"文件管理"开权限 → 返回重试
- **华为/荣耀（HarmonyOS/EMUI）：** 点开文件 → 点击"已下载文件"→ 选择 apk → 同意安全提示 → 点击"继续安装"
- **OPPO/一加（ColorOS）：** 点开文件 → 提示"禁止安装未知来源应用"→ 点击"设置"→ 打开"允许安装未知来源应用"→ 返回重试
- **vivo/iQOO（OriginOS）：** 点开文件 → 输入锁屏密码验证身份 → 点击"坚持安装"

> 各品牌手机第一次装非商店应用都会多几步，装过一次之后下次就顺畅了。

**第 3 步：配置服务器地址**

安装后打开 CC Remote，点击右上角的 ⚙️ 设置图标，填入你电脑的 IP 地址和端口：

| 字段 | 填写 |
|---|---|
| 服务器 IP | 你电脑的局域网 IP（见下文"怎么查电脑 IP"） |
| 端口 | 默认 `11199`，没改过就不用动 |

**怎么查电脑 IP？**
- **Windows：** 按 `Win+R` → 输入 `cmd` 回车 → 输入 `ipconfig` 回车 → 找到"IPv4 地址"（通常是 `192.168.x.x`）
- **Mac：** 系统设置 → 网络 → 当前连接的网络 → 看到 IP
- **Linux：** 终端里输入 `ip addr` 或 `hostname -I`

填好之后回到主页，下拉刷新，如果显示"已连接"就成功了！

### 电脑端（服务端）

服务端需要在你的工作电脑上运行。以下任选一种方式：

**方式 A：一键安装（推荐）**

```bash
# 需要电脑上已安装 Node.js 18+
# 没装的去 https://nodejs.org 下载 LTS 版本

npm install -g cc-remote-server
cc-remote-server
```

**方式 B：手动下载**

```bash
git clone https://gitee.com/romp/cc-remote.git
cd cc-remote/server
npm install
npm start
```

服务端启动后，终端会显示：
```
Server running on http://0.0.0.0:11199
```

> 如果启动报错找不到 `claude` 命令，说明你还没装 Claude Code。先装一下：`npm install -g @anthropic-ai/claude-code`
